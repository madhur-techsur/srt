## Scope

This document specifies the functional behavior of the Simple Request Tracker (SRT) — a React + Java Spring Boot web application that enables users to submit requests and view them in a table. It covers all five PRD features (F0–F4), full database schema, REST API contracts, error catalog, and integration constraints. Authentication, external integrations, pagination, and production security are explicitly out of scope.

## Cross-Cutting Terminology

| Term | Definition |
|------|------------|
| **Request** | A user-submitted record consisting of Name, Title, and Description fields |
| **Requester** | The persona who fills out and submits the request form (Business User) |
| **Viewer** | The persona who reviews submitted requests in the table (Internal User / Engineer) |
| **Submission** | The act of sending a completed form to the backend via HTTP POST |
| **Session** | The lifetime of a running SRT application instance; data persists for the session duration only |
| **In-memory store** | H2 database running embedded in the Spring Boot JVM; data is lost on application restart |
| **API** | The REST interface exposed by the Spring Boot backend at base path `/api` |
| **Client-side validation** | Field checks performed in the React frontend before a network request is made |
| **Server-side validation** | Field checks performed in the Spring Boot backend; authoritative gate |
| **Acknowledgment** | The visual confirmation message shown to the Requester after a successful submission |
| **Empty state** | The UI display shown in the request table when zero requests exist |
